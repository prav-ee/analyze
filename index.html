Data Processing & CI/CD

This project demonstrates a robust data processing pipeline using Python (Pandas), automated code quality checks with Ruff, and continuous integration/deployment via GitHub Actions to publish results to GitHub Pages.

Project Overview

  • Execute.py Fix: Addresses a non-trivial error in the execute.py script for reliable data processing.
  • Data Conversion: Converts data.xlsx into data.csv for efficient data handling.
  • CI/CD Pipeline: Implements a GitHub Actions workflow to lint code, execute the script, and publish the output.

How it Works

  1. The data.xlsx file is converted to data.csv and committed.
  2. A non-trivial error in execute.py is fixed, and the script is updated to read data.csv and output a JSON string.
  3. A GitHub Actions workflow (.github/workflows/ci.yml) is triggered on push to main.
  4. The workflow runs Ruff for code quality and then executes python execute.py > result.json.
  5. Finally, result.json is published as a static asset via GitHub Pages.
